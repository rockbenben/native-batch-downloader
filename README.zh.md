# Native Batch Downloader

一个通过浏览器原生下载通道批量下载文件的 Chrome 扩展。Cookie、User-Agent、Referer 等请求头全部自动携带，无需额外配置。

中文 | [English](README.md)

## 工作原理

点击工具栏图标，把链接粘进去（一行一个），设好间隔和并发数，点 **开始下载** 就行。

底层调用的是 `chrome.downloads.download(url)`，本质就是让浏览器去请求那个 URL 然后保存响应内容，跟你在地址栏输入链接回车是一回事。因为走的是浏览器自己的请求通道，Cookie、UA、Referer 全部自动携带。

## 功能特性

- **原生下载通道** -- 请求经由浏览器本身发出，而非 `fetch` 或 `XMLHttpRequest`，登录态 Cookie 和请求头自动包含。
- **并发与间隔控制** -- 可设置并行下载数量（1-200）及每次下载之间的等待时间（毫秒）。
- **实时进度** -- 总计 / 成功 / 失败 / 等待 四项实时统计，附带进度条。
- **不限文件格式** -- PDF、图片、视频、压缩包、可执行文件——只要是直链，什么都能下。
- **18 种语言** -- AR、DE、EN、ES、FR、IT、JA、KO、NL、PL、PT-BR、RU、TH、TR、UK、VI、ZH-CN、ZH-TW。
- **Manifest V3** -- 现代 Chrome 扩展架构，无后台脚本。

## 安装方法

1. 下载或克隆本仓库。
2. 在 Chrome 中打开 `chrome://extensions/`。
3. 打开右上角的 **开发者模式**。
4. 点击 **加载已解压的扩展程序**，选择本项目文件夹。
5. 工具栏出现扩展图标，点击即可打开弹窗。

## 使用方法

1. 点击工具栏中的扩展图标。
2. 在文本框中粘贴下载直链，**每行一个**。
3. 根据需要调整 **并发数**（默认 10）和 **间隔**（默认 0 ms）。
4. 点击 **开始下载**。
5. 实时查看日志和统计信息。点击 **停止** 可中止任务。

## 注意事项

### URL 必须是直链

如果一个网站是点按钮后通过 JS 动态生成临时下载地址的（比如网盘的"点击下载"），粘贴页面 URL 进去是没用的，需要拿到实际的文件直链。

### 依赖当前浏览器的登录态

能自动带 Cookie 是因为走的是浏览器自己的请求通道，前提是你在浏览器里已经登录了目标网站。如果目标站需要登录但你没登录，该 403 还是 403。

### Chrome 同域名并发限制

Chrome 对同一域名大约只允许 **6 个并发 TCP 连接**，所以即使你设并发 100，同域名下实际同时在传的也就 6 个左右，其余在浏览器内部排队。跨域名则没有这个瓶颈。

### 危险文件类型会触发 Chrome 警告

`.exe`、`.bat` 等可执行文件可能触发 Chrome 内置安全提示，会打断自动下载流程。任何扩展都无法绕过这一限制——`chrome.downloads.acceptDanger()` 调用后 Chrome 仍然会弹出原生确认框。

**解决办法**：打开 `chrome://settings/security`，将安全浏览等级设为 **无保护**，下完后记得改回来。

PDF 等常规文档类型完全不受影响，直接用就行。
